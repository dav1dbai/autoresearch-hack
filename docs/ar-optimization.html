🔍 What it noticed

v1's inner loop scored every iteration on the H100 — even when the agent's edit was a no-op (kernel byte-identical to the last best). Each one burned a ~30s GPU compile+run for zero information.

✏️ What it changed

Added a content-diff guard in solve(): if the workdir didn't actually change, count it stale and skip the score — plus a helper that compares files by size + bytes (ignoring agent.log).

⚡ Why it helps

The inner loop spends its compute budget on real attempts, not re-grading identical kernels — more climbing per unit compute, and stale-detection that reflects actual progress.
