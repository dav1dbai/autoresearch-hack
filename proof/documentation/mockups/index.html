Design principles

  • Hero visual: layered inner curves. One chart, all versions overlaid — early versions pale, late versions indigo, v0 + latest bold. This is the cleanest read for “is the inner loop lifting?” Small multiples are cut.
  • Two-tier story. Layered curves (qualitative) + ΔS(N) bars (quantitative headline). Outer final R* is secondary context.
  • Clean vs hacked on layered curves: hacked versions = thin dashed red, drawn behind clean layers. Same on outer curve + lineage.
  • Uncertainty always visible on S(N) and ΔS(N) (95% CI whiskers from seed replicates).
  • Fixed axes — reward always 0–1; inner step t always 0…K−1 across versions so curves are comparable.
  • No chartjunk — light grid, no 3D, direct labels on key points only.

Hero

1 · Layered inner curves R(N, t)

Full-width overlay on held-out panel: x = inner step, y = reward. Every clean version gets a layer (pale gray → indigo by age). v0 and latest clean are bold anchors. Hacked = dashed red behind the stack. Answers: “does the fan lift?” — the intuitive face of the second derivative.

Primary

3 · Inner slope S(N)

OLS slope of R vs t, averaged over seeds + eval panel. Line + point + CI whiskers. Answers: “how fast does this version learn per iteration?”

Headline metric

4 · Second derivative ΔS(N)

S(N) − S(N−1). Bars from zero; indigo = significant gain, gray = CI crosses zero. Zero reference line emphasized. Answers: “is each generation better at researching?”

Integrity

5 · Train vs held-out gap

Connected dots per version. Large gap + red = overfit flag territory. Pairs with hack detector, not the headline.

Lineage

6 · Archive tree

Parent → child with held-out R on node. Hacked nodes red border. Shows exploration, not just the frontier.
